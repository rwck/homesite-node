Arduino Light Sensor

I still can't get over the fact that you can start with a really simple signal and do a whole bunch of things based on what that signal says.

This project uses an Arduino to get a value from a light sensor. The Arduino sends that value to a simple server running on a laptop. Using websockets and a publish / subscribe approach, the server sends the signal to a Sinatra web app hosted on heroku.

The upshot is that the website shows (in realtime) the value on the light sensor.

Cool.

Or at least, I think so. This is just a tiny insight into what is possible, and I'm actively developing additional functionality.

Safe Landing

This was a joint project with fellow General Assembly student Daisy Kesavan.

Both expats, we decided to build something that would have helped us at the time we moved to Australia.

This app uses three APIs to provide a single view of information people could find useful when considering a move.

  • A map of the place you searched
  • Some pictures from Flickr from the place you searched
  • The details of some meetups held in the place you searched

This is a Rails app with a Backbone front end.

There is significant scope for extra features - the main functionality I'm currently looking to add is a 'Find a Friend' feature that will allow users to search for other users with shared interests.

FMEA

When I was working in recruitment I had a lot of clients who were doing process improvement projects.

One of the key methodologies they use for this is called Lean. And one of the most useful tools from this toolset is called FMEA, or Failure Mode and Effects Analysis.

FMEA is basically a structured approach to helping people and organisations to prioritise the projects they have to do.

I haven't yet found a decent online tool to help people to do this, so I decided to build one.
